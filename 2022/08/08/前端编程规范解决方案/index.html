前端编程规范解决方案

为什么需要编程规范？

团队开发，没有统一的代码标准，呈现出各种乱象

需求：让程序自动处理规范化的内容

  1. 编码规范
  2. git规范

编程规范一：代码检测工具ESLint

ESLint 是 2013年6月创建的一个开源项目，它的目标非常简单，只有一个，那就是提供一个插件化的 javascript 代码检测工具，说白了就是做代码格式检测使用的

eslint 的配置文件：.eslintrc.js

编程规范二：代码格式工具Prettier

  1. 一个代码格式化工具
  2. 开箱即用
  3. 可以直接集成在VSCode中
  5. 保存时让代码直接符合ESLint标准（需要通过一些简单配置）
  6. https://www.prettier.cn/

ESLint与Prettier配合解决代码格式问题

  1. 安装ESLint和Prettier插件
  2. 新建.Prettier文件，该文件为Prettier默认配置文件，配置基本的规则

  1. VS Code配置

    1. Format On Save

    1. 设置一个tab为2个空格

  2. ESLint与Prettier的冲突问题

    1. 方法名和后面的小括号之间，应该有一个空格
    2. 修改 .eslintrc.js'space-before-function-paren': 'off'关闭《方法名后增加空格》的规则

编程规范三：约定式提交规范

Angular团队规范延伸出的 Conventional Commits specification（约定式提交）

其中 类型，必须是一个可选的值，比如：

  1. 新功能：feat
  2. 修复：fix
  3. 文档变更：docs
  4. ….

学习使用 Git Hooks

我们希望：当《提交描述信息》不符合约定式提交规范的时候，阻止当前的提交，并抛出对应的错误提示Git hooks（git 钩子 || git 回调方法）：也就是：git 在执行某个事件之前或之后进行一些其他额外的操作阻止不合规的提交消息，需要使用到 hooks 的钩子函数
